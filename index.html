Generalized Contrastive PCA

Hyperparameter-free contrastive dimensionality reduction for discovering patterns enriched in one condition relative to another.

gcPCA is designed for the common scientific setting in which two high-dimensional datasets must be compared directly. Rather than prioritizing overall variance in a single matrix, it ranks directions by how strongly they are enriched in one condition relative to another. The symmetric v4 formulation is the flagship presentation because it treats both datasets equally while yielding interpretable component scores and eigenvalues.

Why gcPCA

When the scientific question is contrast, vanilla PCA is asking the wrong question.

PCA operates on one dataset at a time and prioritizes total variance. Many experiments, however, are fundamentally comparative: treated versus control, wake versus sleep, baseline versus stimulation, expression A versus expression B. In that setting, the dominant modes of variation can be shared across both conditions and obscure the low-dimensional structure that actually differs.

Single-dataset variance is not the full story

PCA is powerful when one matrix contains the whole question. For paired conditions, it cannot directly ask whether a direction is preferentially enriched in one dataset relative to another.

Shared background structure can dominate

Large common sources of variance, including batch effects, population-wide fluctuations, or shared covariance structure, can overwhelm the condition-specific signal researchers actually want to study.

cPCA introduced the right instinct

cPCA framed the problem as a contrast between conditions, but the method typically requires scanning over a hyperparameter \(\alpha\), yielding multiple candidate embeddings and no intrinsic rule for selecting the correct one.

gcPCA normalizes the contrast

gcPCA reformulates the objective so it emphasizes relative changes in variance instead of favoring high-variance dimensions by default. The result is a practical, hyperparameter-free workflow that is easier to interpret and easier to use.

Visual intuition

Relative enrichment matters more than raw magnitude.

In synthetic settings with strong shared variance, the most informative structure can be low in absolute variance yet clearly enriched in one condition. gcPCA is built to surface that relative change.

Method overview

Four steps from paired datasets to generalized contrastive components.

01

Estimate covariance in both conditions

Start from two matched feature spaces, dataset A and dataset B, and compute the covariance matrices \(C_A\) and \(C_B\).

02

Construct a contrastive objective

Define a direction \(x\) that should score highly when variance is enriched in one dataset relative to the other.

03

Normalize the contrast

Normalize by background variance or total variance, depending on the variant, so the objective reflects relative enrichment rather than raw scale.

04

Solve the eigenproblem and project

Solve the resulting eigenvalue problem, rank the generalized contrastive components, and project each dataset into the low-dimensional space for visualization or downstream analysis.

Mathematical formulation

The flagship v4 objective is symmetric, bounded, and easy to interpret.

Main objective

\[ \max_{x} \frac{x^\mathsf{T}(C_A - C_B)x}{x^\mathsf{T}(C_A + C_B)x}, \quad \text{subject to } \lVert x \rVert_2 = 1 \]

In v4, the numerator measures the contrast between conditions while the denominator normalizes by total variance. This makes the formulation symmetric and naturally suited to comparisons in which both datasets should be treated on equal footing.

Interpretation

  • v4 is symmetric. Swapping A and B flips the sign of the eigenvalues, not the logic.
  • Eigenvalues lie in \([-1, 1]\). The scale is compact and comparable across components.
  • Positive values indicate components enriched in A.
  • Negative values indicate components enriched in B.
  • Values near zero indicate structure shared across both datasets.
Compact summary of gcPCA variants
Variant Objective Interpretation
v2 \(A / B\) Asymmetric ratio useful when B is a background condition.
v3 \((A - B) / B\) Asymmetric relative change with B as the reference.
v4 \((A - B) / (A + B)\) Symmetric contrast and the recommended default for paired conditions.
v2.1 / v3.1 / v4.1 Orthogonal variants Preserve orthogonality in the original feature space when that constraint matters.
Sparse options L1 feature selection Encourage interpretability by selecting compact sets of informative features.

Example results

From controlled synthetic structure to real experimental data.

The method is most compelling when strong shared variance would otherwise dominate the embedding. gcPCA keeps the comparison centered on enrichment, not raw scale.

Synthetic data

PCA follows the highest total variance, which can be largely shared across conditions. cPCA can recover the contrastive manifold, but often only after a manual sweep over \(\alpha\). gcPCA returns components ordered by normalized enrichment, making low-variance condition-specific structure much easier to recover and compare.

Real data: facial expression structure

In the face-expression example presented with the method, gcPCA highlights expression-related structure while discounting shared identity variation. The result is a compact view of the features that differ between conditions without requiring supervised labels or a hyperparameter search.

Comparison to related methods

gcPCA sits between classical PCA and supervised separation methods.

Python package / quick start

Move from paper to working code in a few lines.

The gcPCA toolbox is available as a Python package and as companion code in the main repository. The v4 formulation is a strong default when the two datasets should be compared symmetrically.

Install from PyPI

pip install generalized-contrastive-PCA

PyPI package and source repository are both publicly available.

Quick start

from generalized_contrastive_PCA import gcPCA

model = gcPCA(method="v4", normalize_flag=True)
model.fit(X_A, X_B)

scores_A = model.Ra_scores_
scores_B = model.Rb_scores_
loadings = model.loadings_
eigenvalues = model.gcPCA_values_

The current repository README documents gcPCA(method="v4", normalize_flag=True) followed by fit(X_A, X_B), with outputs such as loadings_, Ra_scores_, Rb_scores_, and gcPCA_values_.
